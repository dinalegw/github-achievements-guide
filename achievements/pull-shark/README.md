# Pull Shark

## Overview

Pull Shark rewards fast, consistent pull request contributions across public repositories. It shows you can spot issues, propose focused changes, and support open source through actionable work.

## Why it matters

This achievement highlights steady collaboration across multiple projects. It helps employers and community members see that you can deliver value beyond a single repository.

## Requirements

* Submit or review at least 10 pull requests.
* Work across at least 3 distinct public repositories.
* Provide clear PR descriptions and follow-up communication.

## Step-by-step

1. Find active repositories with open issues that match your experience.
2. Start with documentation, bug fixes, or tests.
3. Open a pull request with a concise problem statement and solution summary.
4. Respond quickly to review feedback and update your PR.
5. Record each PR so you can track progress across repositories.

## Example

* Fix a typo in one repo, add a small test in another, and update a README in a third.
* Submit PRs with titles like: "Fix broken link in setup docs" or "Add validation for empty form inputs".

## Success checklist

* PR title is clear and scoped.
* Description explains the change and any testing.
* The PR is linked to an issue or clearly labeled.
* Responses are timely and respectful.

## Tips

* Keep PRs small and self-contained.
* Use `good first issue` or `help wanted` labels when possible.
* Avoid large refactors for this achievement; consistency is more valuable than size.
